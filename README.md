技术路线图（初步预案 — 仅供讨论）

下面的时间与任务为粗略估计，实际计划以团队决定为准。

Phase 1 — 原型验证（2–4 周）

目标：跑通一套最小数据通路（单节点）

任务：树莓派 + 传感器（温湿度、加速度/震动、麦克风）→ 本地采集 → 上传云端（HTTP/MQTT）→ 简单阈值报警

Phase 2 — 小规模布点与后台（2–6 周）

目标：3–10 个节点的小场景部署

任务：统一数据格式、轻后台（时间序列存储）、基础可视化面板、异常检测逻辑

Phase 3 — 行业 MVP（6–10 周）

目标：可向潜在客户展示的最小可售产品

任务：场景特化（工厂或电梯）、引入分析模型、报警/运维工作流、客户反馈机制

Phase 4 — 数据闭环与长期研究（持续）

目标：构建持续数据采集、复盘与模型迭代机制，为 HDI 研究提供基础数据
